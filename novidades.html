Domingo, 06 de Abril de 2026 v12.0

Pipeline Engine + Cadence DB-Driven + Intelligence Fix

Transição de deals com gates server-side, cadências migradas de código hardcoded para banco de dados, edge function de inferência corrigida e polish visual na interface.

⚡
Pipeline Transition Engine (D2)
Core · Pipeline
O cockpit era um viewer passivo — mostrava deals mas não podia transicioná-los entre etapas. Agora o operador move deals com gates obrigatórios, kill switches e audit trail completo, tudo validado server-side.
O que mudou
transition-ui.js — botão "Avançar Etapa" com modal de gates e confirmação
transition-validator — edge function com 11 gates e 5 kill switches (server-side)
Grafo configurável — transições permitidas armazenadas no banco, não hardcoded
Audit trail — toda transição registrada com operador, timestamp e evidência
PIPELINE GATES KILL SWITCHES EDGE FUNCTION
🔄
Cadence System — Templates DB-Driven
Core · Cadência
As cadências de follow-up estavam hardcoded no JavaScript — qualquer mudança exigia deploy. Agora os templates vivem no Supabase e são editáveis sem deploy, com suporte a persona e tier do deal.
O que mudou
Código legado removido — 250+ linhas de templates hardcoded substituídos por query ao banco
5 cadências seed — Titan Hot, Titan Warm, Builder Standard, Builder Express, Executor Intensive
Canal email eliminado — cadências agora usam WhatsApp, Ligação, Instagram, LinkedIn e Tarefa
Init chain sequencial — templates carregam primeiro, depois enrollments e steps
Dialer queues — tabela pronta para filas inteligentes de ligação (Fase 3)
WA Templates — catálogo de 5 templates WhatsApp para envio automatizado (Fase 2)
CADÊNCIA DB-DRIVEN SUPABASE 5 FASES
🔧
Elucy Inference — Payload Fix
Bug Fix · Edge Function
O ELUCI Report e Análise de Mercado retornavam erro 400 — o frontend enviava campos com nomes diferentes do esperado pela edge function. Agora aceita ambos os formatos sem breaking change.
Correções
requestType → request_type — edge function agora aceita ambos
userMessage fallback — mensagem pré-montada usada quando campos separados ausentes
dealId → deal_id — mapeamento bidirecional sem breaking change
BUG FIX EDGE FUNCTION INFERENCE

Ambient Design — Dot Grid + Glassmorphism
Design System · Cockpit + Landing
O cockpit e a landing tinham fundo sólido sem profundidade. Inspirado no padrão de pontos do G4 OS, adicionamos dot grid amber, orbs flutuantes animados e glassmorphism nos painéis — criando coerência visual entre todas as páginas do sistema.
O que mudou
Cockpit: dot grid radial amber (26px) com mask elíptica + 3 orbs animados (cockpitOrb keyframe, 24s)
Landing: dot grid amber (28px) com orbs migrados de azul Apple para amber Elucy
Topbar: glass profundo — blur 32px saturate 200%, borda amber sutil
Sidebar: glass 50% opacidade, blur 28px, inset amber glow
Cards .card: glass real — blur 20px saturate 180%, highlight interno
Deal cards: backdrop-filter + box-shadow por profundidade
COCKPIT LANDING CSS GLASS AMBIENT

Tour de Onboarding — Revisão Completa
UX · Cockpit
O tour original tinha duplo render (dois tooltips ao mesmo tempo), seletores CSS quebrados por posição DOM, tooltip saindo fora da tela no último passo e abas inexistentes como "DM War Room". Reescrevemos do zero.
Correções
Guard duplo: _authHandled previne que getSession + onAuthStateChange disparem checkApproval duas vezes
Seletores infalíveis: trocados de .ntab:nth-child(N) para [onclick*="setScreen('tab"] — não depende de posição DOM
Clamp de viewport: tooltip nunca sai da tela — flip automático bottom→top se transbordar
9 passos reais: Home, Deals, Pipeline, Tasks, Reports, Pre-Vendas, Intel, Playbooks, Perfil
CSS amber: cor das bordas e dots corrigida para #f59e0b (era roxo herdado)
9 PASSOS CLAMP VIEWPORT BUG FIX
